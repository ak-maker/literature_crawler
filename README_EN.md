## Scoring System (0-100)

Relevance-first design with high weight on code/demo availability:

| Component | Max | Description |
|-----------|-----|-------------|
| **Relevance** | 45 | Three-tier keyword matching: critical(+8/+4) high(+5/+2) medium(+2/+1), title weighted higher than abstract |
| **Bonus** | 30 | +15 open-source code, +15 real-world experiment/demo |
| **Venue** | 15 | Tier1=15 (ICRA/IROS/RA-L/RSS/CoRL/NeurIPS/ICML/ICLR), Tier2=10, Tier3=5 |
| **Impact** | 10 | Citation count / paper age, percentile ranking |

Design principle: irrelevant top-venue papers won't rank high; papers with code + demo get a significant boost.

## Code/Demo Detection

Three-layer detection:

1. **Abstract regex** â€” Match `github.com`, `code available`, `real-world`, `hardware experiment`, etc. in the paper abstract
2. **arXiv comment field** â€” Many authors write "code at github.com/..." in the comment
3. **arXiv page scraping** â€” For papers with an arXiv ID, fetch the `arxiv.org/abs/XXXX` page and extract GitHub/GitLab links and demo keywords from the full HTML

Limitation: if code/demo links only appear in the PDF body or footnotes (not in abstract or arXiv page), they will be missed.

## Citation Graph

Queries Semantic Scholar for citation relationships between high-scoring papers and generates a Research Rabbit-style interactive network graph.

### Usage

```bash
# First run the pipeline to generate papers.json, then run graph.py
python pipeline.py searches/uav_ugv.yaml
python graph.py output/uav_ugv/papers.json

# Custom: minimum score 30, max 100 nodes
python graph.py output/uav_ugv/papers.json --min-score 30 --max-nodes 100

# Specify output path
python graph.py output/uav_ugv/papers.json -o output/uav_ugv/my_graph.html
```

Open `citation_graph.html` in a browser.

### Interaction

- **Scroll to zoom**, **drag to move** nodes and canvas
- **Hover over nodes**: shows paper details (title, year, score, venue, citations, paper link, code link)
- **Arrow direction**: A â†’ B means A cites B

### Visual Encoding

| Element | Meaning |
|---------|---------|
| Node size | Higher score = larger node |
| Node color | Method category (RL=red, MPC=blue, Formation=green, Path Planning=orange, GNN=deep orange, Swarm=pink, Communication=brown, SLAM=teal, etc.) |
| Gold border | Has Code + Demo |
| Green border | Code only |
| Pink border | Demo only |
| Semi-transparent node | No citation relationships with other papers (isolated) |

### How It Works

1. Load `papers.json`, filter by `--min-score`, take top `--max-nodes`
2. For each paper, query S2 API for its `references` (reference list)
3. Check if each reference is also in our paper set (matching by S2 paper ID or ArXiv ID)
4. If A's references include B, draw an edge A â†’ B
5. Generate interactive HTML with `pyvis` (based on vis.js), using ForceAtlas2 force-directed layout

## How the Pipeline Works

### Step 1: Fetch Papers

For each keyword in every `search_queries` block, **two sources are queried simultaneously**:

**arXiv API** (`http://export.arxiv.org/api/query`)
- Free, no key required
- Keywords are split into AND-connected terms: `"UAV UGV coordination"` â†’ `all:UAV AND all:UGV AND all:coordination`
- Can filter by arXiv categories (e.g., `cs.RO`, `cs.MA`) to exclude irrelevant fields
- Returns XML (Atom feed), parsed for: title, abstract, authors, year, comment (often contains venue info), PDF link
- Rate limit: 3-second interval between requests

**Semantic Scholar API** (via ai4scholar.net proxy, compatible with official S2 API)
- Endpoint: `https://ai4scholar.net/graph/v1/paper/search`
- Auth: `Authorization: Bearer <key>`
- Searches the raw keyword string; S2 performs semantic matching internally (smarter than arXiv's pure keyword matching)
- Returns JSON with **more data than arXiv**: `citationCount`, `venue` (conference/journal name), `openAccessPdf`, `fieldsOfStudy`, `publicationTypes`
- Rate limit: 0.5s with key, 3s without

5 search blocks x 5-8 keywords x 2 sources â‰ˆ 1200+ raw results

### Step 1.5: Deduplication + Code/Demo Detection

**Deduplication**: Normalize titles (strip spaces, punctuation, case) and match. When both sources find the same paper, merge metadata (keep higher citation count, more complete venue info, etc.). 1230 â†’ 712 papers.

**Code detection** (three layers):
1. Abstract regex: match `github.com`, `gitlab.com`, `code available`, `open source`
2. arXiv comment field: many authors write "code at github.com/..."
3. arXiv page scraping: for papers with an arXiv ID, GET `arxiv.org/abs/XXXX` and search the HTML for GitHub/GitLab links

**Demo detection**: Match keywords in abstract/comment/arXiv page:
`real-world`, `real robot`, `hardware experiment`, `physical experiment`, `field test`, `outdoor experiment`, `deployed`, `flight experiment`, etc.

Limitation: if code/demo info only appears in PDF body footnotes, it will be missed.

### Step 2: Scoring

Each paper receives 4 sub-scores, summed and sorted in descending order:

| Sub-score | Max | Algorithm |
|-----------|-----|-----------|
| **Relevance** | 45 | Three-tier keyword matching. `critical` ("UAV-UGV", "aerial-ground"): title +8 / abstract +4. `high` ("UAV", "coordination"): title +5 / abstract +2. `medium` ("reinforcement learning", "path planning"): title +2 / abstract +1. Capped at 45 |
| **Bonus** | 30 | Open-source code +15, real-world experiment/demo +15 |
| **Venue** | 15 | Regex match on venue field. Tier1 (ICRA/IROS/RA-L/RSS/CoRL/NeurIPS/ICML/ICLR/IJRR/T-RO) = 15, Tier2 (AAMAS/CDC/ICUAS) = 10, Tier3 (Workshop) = 5, arXiv preprint = 2 |
| **Impact** | 10 | `citations / paper_age` percentile within all results x 10 |

### Step 3: Analysis

- **High-frequency keywords**: Extract bigrams and unigrams from titles + abstracts, filter stopwords, rank by frequency
- **Top authors**: Ranked by paper count + average paper score
- **Method categories**: 18 predefined categories (RL, Diffusion, GNN, Transformer, MPC, Lyapunov, Formation Control, Path Planning, SLAM, Communication, Swarm, etc.), each paper assigned to matching categories by abstract keywords
- **Method evolution**: Year x method keyword matrix showing which methods are growing in which years

### Step 4: Generate Reports

Output to `output/<config_name>/`:
- `papers.csv`: Excel-friendly, sorted by total score, with score breakdown
- `papers.json`: Structured data
- `report.md`: Markdown report (Top 50 table, collapsible method categories with paper links, evolution timeline, Code/Demo lists, etc.)
- `config_used.yaml`: Config snapshot (`yaml.dump` auto-generated, keys alphabetically sorted, no comments, for reproducibility only)
