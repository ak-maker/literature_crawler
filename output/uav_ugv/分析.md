  搜索原理详解

  整个 pipeline 分 5 步：

  Step 1: 抓取

  对 YAML 配置里的每个关键词，同时查两个来源：

  arXiv API (export.arxiv.org/api/query)
  - 把 "UAV UGV coordination" 拆成 all:UAV AND all:UGV AND all:coordination
  - 加分类过滤如 cat:cs.RO OR cat:cs.MA
  - 返回 XML (Atom feed)，解析标题、摘要、作者、年份、comment（含 venue 信息）
  - 每次最多 50 篇，3 秒间隔

  Semantic Scholar API (via ai4scholar.net proxy)
  - 直接搜 query="UAV UGV coordination"
  - 返回 JSON，比 arXiv 多：citationCount、venue、openAccessPdf、fieldsOfStudy
  - 每次最多 100 篇

  5 个 block × 5-8 个关键词 × 每个关键词 arXiv+S2 = ~1230 条原始结果

  Step 1.5: 去重 + 代码检测

  - 去重：按标题 normalize（去掉空格/标点/大小写）匹配，合并两个来源的元数据
  - 代码检测：
    a. 先在摘要里 regex 匹配 github.com、code available 等
    b. 对有 arXiv ID 的论文，爬取 arxiv.org/abs/XXXX 页面找 GitHub 链接
    c. Demo 检测：匹配 real-world、hardware experiment、field test、deployed 等

  Step 2: 打分

  每篇论文得 4 个子分：

  ![image-20260310231820831](C:\Users\Logan\AppData\Roaming\Typora\typora-user-images\image-20260310231820831.png)

  按 total = R + B + V + I 降序排序。

  Step 3: 分析

  - 高频关键词：从标题+摘要提取 bigram/unigram，过滤 stopwords
  - Top 作者：按论文数 + 平均分排序
  - 方法分类：18 个预定义类别（RL、MPC、Formation、Diffusion...），每篇论文按摘要关键词归类
  - 方法演化：按年份 × 方法关键词矩阵统计趋势

  Step 4: 输出

  生成 CSV + JSON + Markdown 报告。
